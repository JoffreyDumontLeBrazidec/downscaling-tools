## 2) Predictions (25 files) Then Eval (Automatic Chain)

This route is the canonical checkpoint path for the mature `o96 -> o320` flow. It is not the default front door for `o320 -> o1280`.

Use:

```bash
/home/ecm5702/dev/downscaling-tools/eval/jobs/codex_eval_predictions --ckpt-id <CKPT_ID> [--run-id <RUN_ID>]
```

Examples:

```bash
# Auto run-id: manual_<4 alphanumeric chars>
/home/ecm5702/dev/downscaling-tools/eval/jobs/codex_eval_predictions --ckpt-id 4a5b2f1b24b84c52872bfcec1410b00f

# Explicit run-id (recommended)
/home/ecm5702/dev/downscaling-tools/eval/jobs/codex_eval_predictions --ckpt-id 4a5b2f1b24b84c52872bfcec1410b00f --run-id manual_abcd

# Short suffix is normalized automatically to manual_<suffix>
/home/ecm5702/dev/downscaling-tools/eval/jobs/codex_eval_predictions --ckpt-id 4a5b2f1b24b84c52872bfcec1410b00f --run-id abcd
```

What it does:
- Starts `autopilot_predictions.py` in background.
- Submits dependent jobs:
  1. GPU prediction job (`predict25_<RUN_ID>.sbatch`) to create 25 files:
     - `predictions_YYYYMMDD_stepXXX.nc`
     - each file contains all 10 members
  2. Eval job (`eval25_<RUN_ID>.sbatch`) that runs automatically after predictions complete.

Generated scripts:
- `/home/ecm5702/perm/eval/<RUN_ID>/jobs/predict25_<RUN_ID>.sbatch`
- `/home/ecm5702/perm/eval/<RUN_ID>/jobs/eval25_<RUN_ID>.sbatch`

Where outputs go:
- Predictions: `/home/ecm5702/perm/eval/<RUN_ID>/predictions/`
- Eval run outputs: `/home/ecm5702/perm/eval/<RUN_ID>/eval/*`

Where to check:
- Background log: `/home/ecm5702/perm/eval/<RUN_ID>/logs/autopilot_predictions_background.log`
- PID: `/home/ecm5702/perm/eval/<RUN_ID>/logs/autopilot_predictions.pid`
- State: `/home/ecm5702/perm/eval/<RUN_ID>/logs/autopilot_predictions_state.json`

### Important: 9km / Heavy Checkpoints

- Some checkpoints can OOM on 1 GPU.
- Recommended baseline for heavy checkpoints: **4 GPUs**.
- QoS guidance:
- `dg` max walltime is `00:30:00`. `ng` max walltime is `48:00:00`.
- Use `--predict-qos dg` only for short tests (<= 00:30:00).
- For longer runs, use `--predict-qos ng` and keep `--predict-time` <= 48:00:00.
- Recommended commands:

```bash
# Short test
/home/ecm5702/dev/downscaling-tools/eval/jobs/codex_eval_predictions \
  --ckpt-id <CKPT_ID> \
  --run-id manual_9km4g_test \
  --predict-qos ng \
  --predict-time 00:20:00 \
  --predict-gpus 4 \
  --predict-cpus 32 \
  --predict-mem 768G

# Longer/full run
/home/ecm5702/dev/downscaling-tools/eval/jobs/codex_eval_predictions \
  --ckpt-id <CKPT_ID> \
  --run-id manual_9km4g_full \
  --predict-qos ng \
  --predict-time 02:00:00 \
  --predict-gpus 4 \
  --predict-cpus 32 \
  --predict-mem 768G
```

- `manual_inference.prediction.predict` now supports distributed model-comm-group inference (Slurm/Torch distributed env), so multi-rank GPU launches can shard model work without changing output format.
- The prediction loader now uses checkpoint-native dataloader config (instead of hardcoded O320 assumptions), which is required for non-O320 checkpoints.

## 3) O320 -> O1280 Manual Eval Flow (Proxy -> Continue-Full)

Use the lane helper:

```bash
CHECKPOINT_PATH=<CKPT_PATH> PHASE=proxy \
  bash /etc/ecmwf/nfs/dh2_home_a/ecm5702/dev/downscaling-tools/eval/jobs/templates/submit_o320_o1280_manual_eval_flow.sh
```

Continue the same run after a passing proxy:

```bash
CHECKPOINT_PATH=<CKPT_PATH> PHASE=continue-full RUN_ID_OVERRIDE=<same_run_id> \
  bash /etc/ecmwf/nfs/dh2_home_a/ecm5702/dev/downscaling-tools/eval/jobs/templates/submit_o320_o1280_manual_eval_flow.sh
```

What it does:
- renders run-local submit copies under `/home/ecm5702/dev/jobscripts/submit/<YYYYMMDD>/`
- rebuilds truth-aware bundles under `<RUN_ROOT>/bundles_with_y`
- launches strict prediction from that rebuilt bundle root
- chains one-date local plots, spectra, and TC evaluation

Do not point strict `o320 -> o1280` prediction directly at `/home/ecm5702/hpcperm/data/input_data/o320_o1280/idalia`; that is the raw source GRIB tree, not the strict prediction input root.
